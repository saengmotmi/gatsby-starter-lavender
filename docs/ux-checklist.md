# SSG UX Checklist

이 문서는 이 저장소를 테스트베드로 운영하기 위한 기준입니다.
목표는 "정적 블로그지만 앱 수준의 사용성"입니다.

## 핵심 UX

- [ ] 홈 첫 렌더가 빠르다. (모바일 LCP < 2.5s)
- [ ] 홈에는 요약 데이터만 직렬화한다. (본문/TOC는 상세에서 로드)
- [ ] 포스트 이전/이후 이동 시 댓글 대상이 현재 포스트로 동기화된다.
- [ ] TOC 앵커 클릭, 해시 URL, 뒤로가기 시 스크롤 동작이 일관적이다.
- [ ] 코드블록/인라인 코드 스타일이 가독성과 일관성을 만족한다.
- [ ] 다크모드 전환 시 FOUC 없이 상태가 유지된다.
- [ ] 모바일/데스크톱 타이포/여백 가독성이 유지된다.

## 탐색성

- [ ] 검색(제목/요약/태그) 반응이 즉각적이다.
- [ ] 태그/카테고리 필터 상태를 URL로 공유할 수 있다.
- [ ] 이전/다음 글 네비게이션이 올바르게 동작한다.
- [ ] RSS, sitemap, canonical, OG/Twitter 메타가 정확하다.

## 안정성

- [ ] 404와 리다이렉트 정책이 명확하다.
- [ ] 서비스워커/캐시 영향 없이 새 배포를 사용자가 수신한다.
- [ ] 키보드 탐색/헤딩 구조/명도 대비 등 접근성을 충족한다.
- [ ] CI에서 `typecheck + build + RSS`를 강제한다.

## SSG 예산(Budget)

- [ ] 홈 데이터 payload: `<= 50KB` (권장, gzip 기준)
- [ ] 상세 페이지 데이터 payload: `<= 150KB` (권장)
- [ ] 초기 JS payload(엔트리+핵심 청크): 지속 모니터링
- [ ] 폰트 전송량: 초기 경로 기준 최소화

## 운영 규칙

1. 기준 미충족 항목이 생기면 "퍼포먼스 이슈"를 생성한다.
2. 이슈 본문에 체크리스트 항목을 그대로 복사해 상태를 관리한다.
3. 측정 결과는 이슈 코멘트로 누적한다. (날짜/커밋/측정 방식 포함)
